# oneai-guide

A user-friendly guide to the platform's settings, written for the internal product / design / engineering team. The full parameter reference lives in `reference/`; this guide sits alongside it as the readable companion.

## Tone & structure rules

- **Concepts, not parameters.** Headers and bullets are named after concepts. Parameter names go in parens or as sidenotes — never as section headers.
- **Only user-facing stuff.** If a knob is platform-tuned, internal, or never touched by users, leave it out. Don't recreate the reference "but nice."
- **Concise.** ~2 pages per category. Short paragraphs. Trim ruthlessly.
- **List the things you configure.** For each section, after a brief intro, use a bullet list of *things you configure* (concept name in bold, one-line explanation, params in parens).
- **Ask when unsure.** If you don't understand a concept or aren't sure whether it's user-facing, ask before writing.
